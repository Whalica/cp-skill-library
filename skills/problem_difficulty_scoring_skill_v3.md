# 题目难度评分 Skill v3

## 目标

用于脱离官方 rating，对算法竞赛题目进行结构化难度评价，并同时给出两类难度：

1. **公共难度**
   - 面向普通竞赛群体的题目客观难度。
   - 更接近 CF / AtCoder / Nowcoder 中常见的公共难度分层。
   - 用于估算题目本身大约处于什么 rating 档位。

2. **目标人群体感难度**
   - 面向指定水平选手的赛时体感。
   - 例如目标人群为 CF 4000 左右选手时，大量 1200~2000 题的体感会被显著压低。
   - 用于评估该目标人群做这题时的实际负担。

核心原则：

```text
公共难度 ≠ 目标人群体感难度
```

输出时必须同时给出二者，不能用其中一个替代另一个。

---

## 可配置项

使用本 Skill 时，可以直接修改本节配置。

```yaml
target_group:
  name: "CF 1400~1600 左右普通竞赛选手"
  rating: 1400~1600

public_difficulty_reference:
  name: "普通竞赛群体"
  rating_range: "CF 1200~2200 混合群体"

output_requirements:
  include_public_difficulty: true
  include_target_group_difficulty: true
  include_cf_rating_bucket: true
  include_ac_vs_proof_difficulty: true
```

说明：

- `target_group.rating` 可以改成任意目标人群 rating，例如 `1400`、`1600`、`2200`、`3000`、`4000`。
- 公共难度不随 `target_group.rating` 直接改变。
- 目标人群体感难度会根据 `target_group.rating` 重新校准。

若用户未指定目标人群，默认使用：

```text
CF 1400~1600 左右普通竞赛选手
```

## 评分范围

所有基础维度分数均为 `0.0 ~ 10.0`，保留一位小数。

分数含义：

- `0.0`：几乎没有难度
- `2.0`：较低难度
- `5.0`：中等难度
- `8.0`：较高难度
- `10.0`：极高难度

最终分也控制在 `0.0 ~ 10.0` 内，保留一位小数。

## 公共难度与目标人群体感

### 公共难度

公共难度回答：

```text
这题作为一道竞赛题，大约相当于什么难度？
```

它主要考虑：

- 普通竞赛群体的平均赛时 AC 难度；
- 题目的客观结构；
- 做法是否常见；
- 实现是否繁琐；
- 错路是否多；
- 题面是否难建模；
- 数据范围是否强提示；
- 是否能通过小样例发现规律。

公共难度最终会映射到 CF 合法档位：

```text
800, 900, ..., 3500, 3500+
```

### 目标人群体感难度

目标人群体感回答：

```text
指定水平的选手做这题时，主观上会觉得多难？
```

它会随目标人群水平变化。

例如同一道公共难度 1900 的题：

| 目标人群 | 体感 |
|---|---:|
| CF 1200 | 很难，可能 `7.0~8.5/10` |
| CF 1600 | 偏难，可能 `5.0~6.5/10` |
| CF 2200 | 常规，可能 `3.0~4.5/10` |
| CF 3000 | 较简单，可能 `1.0~2.5/10` |
| CF 4000 | 热身题，可能 `0.0~1.0/10` |

注意：

```text
目标人群体感难度不能反向修改公共难度。
```

## 基础评价维度

每个维度需要分别给出：

- 公共难度分
- 目标人群体感分

若目标人群就是默认普通群体，两组分数通常接近；若目标人群很强或很弱，两组分数可能差异很大。

---

### 1. 算法 / 模板门槛

评价是否需要专门算法、数据结构或固定模板。

没有明显模板时，不纳入基础分，不给 `0`。

参考：

- `0~2`：基础语法、排序、gcd、前缀和、简单枚举
- `3~5`：二分、BFS、DFS、简单 DP、并查集、Fenwick
- `6~8`：线段树、树形 DP、最短路变形、字符串算法、网络流基础
- `9~10`：后缀自动机、点分治、支配树、高级图论、高级数据结构

注意：

- 裸模板题要区分“会模板的人”和“不会模板的人”的体感差异。
- 若模板只是辅助，不能把模板门槛当作主导难点。
- 若存在无模板解，应按赛时合理可想到的最简单正解评分。

---

### 2. 结论发现难度

评价关键性质、观察、构造方式在赛时被发现的难度。

此项优先评价“发现成本”，而不是结论本身是否少见。

参考：

- `0~2`：结论几乎直接可见，样例或题面强提示
- `3~5`：需要手推、画图、试小样例，但较自然
- `6~8`：需要针对性推导，不容易通过普通小样例看出
- `9~10`：核心结论非常隐蔽，接近“想不到就做不了”

重要区分：

```text
结论陌生 ≠ 结论难发现
```

若一个结论看起来陌生，但能通过 3~5 个小样例自然发现，则本项不应给高分。

目标人群体感中，若该人群对该类结论非常熟悉，应显著下调。

---

### 3. 思维转化难度

评价从题面到正解需要经过多少层转化，以及逻辑推理链是否自然。

参考：

- `0~2`：题面几乎直接给做法
- `3~5`：需要一到两层常规转化
- `6~8`：需要多层推理、反向思考或关键建模
- `9~10`：推理链很深，切入点极难自然想到

注意：

- 若本项打高的原因与“结论发现难度”完全相同，应进行难点去重。
- 单点观察题不能在结论发现、思维转化、建模三个维度中重复计费。

---

### 4. 实现与边界风险

评价代码实现成本，而不只看代码长度。

考虑因素：

- 代码量
- 状态维护复杂度
- 函数数量
- 分类讨论数量
- 下标处理
- 取模 / 溢出
- 多测清空
- 特殊边界
- 调试成本
- 出错概率

参考：

- `0~2`：几行判断或简单循环，几乎无坑
- `3~5`：普通实现，有少量状态或边界
- `6~8`：实现较长，状态多，容易写错
- `9~10`：大型模拟、复杂数据结构、多层分类讨论，debug 成本很高

强制规则：

- 若公共实现复杂度 `>= 7.0`，公共最终分通常不应低于 `5.5`。
- 若公共实现复杂度 `>= 8.0` 且边界密集，公共最终分通常不应低于 `6.0`。
- 几何、交互、大模拟、复杂构造应额外关注验证难度。

目标人群体感中，强选手的实现风险可下降，但不能直接归零。对于长代码、大模拟、交互、几何，即使目标人群很强，也应保留一定实现风险。

---

### 5. 题意阅读与建模难度

评价理解题意、整理操作、抽象状态、建立数学或算法模型的难度。

可拆成两部分思考：

- 题面阅读难度：题面是否长、绕、角色多、时序复杂、翻译腔严重
- 模型抽象难度：是否难以把题面转化成可计算模型

参考：

- `0~2`：题意直接，模型显然
- `3~5`：题面较长或状态需要整理
- `6~8`：操作复杂，抽象模型不明显
- `9~10`：题意本身非常绕，建模是主要门槛

---

### 6. 错误方向诱导性 / 试错成本

评价题目是否容易把选手引向错误做法，以及排除错误方向的成本。

参考：

- `0~2`：做法方向自然，错路少
- `3~5`：有一些常见误区
- `6~8`：错路很多，容易写假算法
- `9~10`：正解反直觉，错误方向极强

典型加分场景：

- 看起来能贪心，但贪心全错
- 看起来能 DP，但状态爆炸
- 看起来要数据结构，其实是结论
- 样例弱，假做法容易过样例
- 存在大量伪规律
- 正解很短但方向极不自然

## 基础分计算

对纳入的基础维度加权平均。

默认权重：

| 维度 | 权重 |
|---|---:|
| 算法 / 模板门槛 | 0.20 |
| 结论发现难度 | 0.25 |
| 思维转化难度 | 0.20 |
| 实现与边界风险 | 0.20 |
| 题意阅读与建模难度 | 0.15 |
| 错误方向诱导性 / 试错成本 | 0.15 |

若某项“不纳入”，则去掉该项权重，并对剩余权重归一化。

公式：

```text
基础分 = Σ(维度分 × 维度权重) / Σ(纳入维度权重)
```

公共难度与目标人群体感难度分别计算基础分。

## 赛时修正项

基础分反映题目结构难度，最终分需要根据赛时体感修正。

```text
最终分 = clamp(基础分 + 修正项总和, 0.0, 10.0)
```

修正项总和一般控制在 `-2.0~+2.0`，极端情况可突破，但必须解释。

公共难度与目标人群体感难度分别应用修正项。同一个修正项对二者的数值可以不同。

---

### 1. 规律可发现性修正

| 情况 | 修正 |
|---|---:|
| 3~5 个小样例即可明显发现规律 | `-0.8~-1.2` |
| 小样例有一定提示，但不稳定 | `-0.3~-0.6` |
| 小样例无明显提示 | `0` |
| 小样例明显误导 | `+0.3~+0.8` |

---

### 2. 约束提示强度修正

| 情况 | 修正 |
|---|---:|
| 约束非常明显地提示正解方向 | `-0.5~-1.0` |
| 约束有一定提示 | `-0.2~-0.5` |
| 约束无明显提示 | `0` |
| 约束具有迷惑性 | `+0.3~+0.8` |

---

### 3. 难点去重修正

| 情况 | 修正 |
|---|---:|
| 两个维度明显重复计费 | `-0.3~-0.6` |
| 三个及以上维度来自同一核心观察 | `-0.6~-1.0` |
| 多个难点彼此独立 | `0` |

---

### 4. 多解路径修正

| 情况 | 修正 |
|---|---:|
| 有明显更简单做法 | `-0.5~-1.0` |
| 有暴力 / 打表 / 对拍辅助发现 | `-0.3~-0.8` |
| 正解路径唯一且很窄 | `+0.3~+0.8` |
| 只看到复杂官方解，民间解未知 | 暂不修正，并说明不确定 |

原则：

```text
评分应基于赛时合理可想到的最简单正解，而不是最正式、最优雅或官方给出的做法。
```

---

### 5. 可验证性 / 对拍难度修正

| 情况 | 修正 |
|---|---:|
| 很容易写暴力对拍 | `-0.3~-0.6` |
| 普通 | `0` |
| 很难验证 | `+0.3~+0.8` |

---

### 6. 样例提示修正

| 情况 | 修正 |
|---|---:|
| 样例强烈提示正解 | `-0.3~-0.8` |
| 样例覆盖主要边界 | `-0.2~-0.4` |
| 样例普通 | `0` |
| 样例弱或误导 | `+0.2~+0.6` |

---

### 7. 数据强弱修正

若评价目标是“赛时通过难度”，可以考虑数据强弱。

| 情况 | 修正 |
|---|---:|
| 数据明显弱，假做法可过 | 赛时通过难度下调 |
| pretest 弱但 system test 强 | 说明赛时风险，不直接下调理论难度 |
| 数据强，卡边界充分 | 实现与边界风险上调 |
| 数据未知 | 不修正 |

必须区分：

- 理论正解难度
- 赛时通过难度

## 目标人群体感校准

当目标人群明显强于公共难度对应群体时，体感分应显著下降。  
当目标人群明显弱于公共难度对应群体时，体感分应显著上升。

先得到公共 CF 估分，再与目标人群 rating 比较：

```text
gap = 目标人群 rating - 公共 CF 估分
```

经验校准：

| gap | 目标人群体感 |
|---:|---:|
| `<= -800` | 极难，可能 `8.0~10.0` |
| `-800~-500` | 很难，可能 `7.0~8.5` |
| `-500~-300` | 偏难，可能 `6.0~7.5` |
| `-300~-100` | 略难，可能 `4.5~6.0` |
| `-100~+100` | 正常匹配，接近公共最终分 |
| `+100~+300` | 略简单，可能 `3.0~4.5` |
| `+300~+600` | 简单，可能 `1.5~3.5` |
| `+600~+1000` | 很简单，可能 `0.5~2.0` |
| `>= +1000` | 热身 / 秒题，通常 `0.0~1.0` |

注意：

- 此表是校准参考，不是硬公式。
- 若题目有极高实现风险、交互、几何或长代码，即使 gap 很大，也不能随意降到 `0`。
- 若题目是目标人群极其熟悉的套路，可进一步下调。
- 若题目是冷门专题或极端实现题，可保留更高体感。

### CF 4000 目标人群特殊规则

若目标人群为 CF 4000 左右选手：

- 公共难度 `<= 2000` 的题，多数体感应在 `0.0~2.0`
- 公共难度 `2000~2600` 的题，多数体感应在 `1.0~3.0`
- 公共难度 `2600~3200` 的题，多数体感应在 `2.0~5.0`
- 公共难度 `3200~3500+` 的题，才可能达到 `4.0+`

但以下题型应保留体感：

- 长实现
- 复杂几何
- 交互
- 输出构造验证困难
- 大模拟
- 需要大量细节的工程型数据结构
- 极冷门专题

## 瓶颈规则

### 1. 单点门槛下限

若存在不可绕过的核心门槛，公共最终分通常不应低于：

```text
核心门槛分 - 0.5
```

目标人群体感是否应用此下限，取决于该目标人群是否会被该门槛卡住。

---

### 2. 短代码规律题上限

若同时满足：

- 无明显算法 / 模板门槛
- 实现短
- 边界少
- 核心规律可通过 3~5 个小样例发现
- 没有强误导方向

则公共最终分通常不应超过：

```text
4.5
```

若规律极其明显，可进一步限制在：

```text
3.5~4.0
```

目标人群很强时，此类题体感可进一步降到 `0.0~1.5`。

---

### 3. 实现题下限

若题目主思路常规，但实现复杂度极高：

- 公共实现复杂度 `>= 7.0`：公共最终分通常不低于 `5.5`
- 公共实现复杂度 `>= 8.0` 且边界密集：公共最终分通常不低于 `6.0`

目标人群体感中，强选手可以下调，但仍需保留实现压力。

---

## 难度类型分离

输出时应尽量区分：

```text
赛时 AC 难度
严格证明难度
实现落地难度
题解表达难度
```

说明：

- 赛时 AC 难度：选手在比赛中做出并通过的难度，公共 CF 估分主要参考它。
- 严格证明难度：把正确性完整讲清楚的难度。
- 实现落地难度：知道做法后写出并调对的难度。
- 题解表达难度：把题目写成清楚题解的难度。

## CF 估分规则

### 1. 连续估分

先用公共最终分给出连续估计：

```text
连续估分 = 800 + 200 × 公共最终分
```

目标人群体感分不直接映射为公共 CF 估分。

---

### 2. CF 档位化

最终展示时必须按 CF 实际题目难度档位化。

合法档位为：

```text
800, 900, 1000, 1100, 1200, 1300, 1400, 1500,
1600, 1700, 1800, 1900, 2000, 2100, 2200, 2300,
2400, 2500, 2600, 2700, 2800, 2900, 3000, 3100,
3200, 3300, 3400, 3500, 3500+
```

档位化规则：

1. 若连续估分 `< 850`，档位为 `800`。
2. 若连续估分在 `[850, 3450)`，四舍五入到最近的整百。
3. 若连续估分在 `[3450, 3550)`，档位为 `3500`。
4. 若连续估分 `>= 3550`，档位为 `3500+`。

可同时给出区间：

```text
公共 CF 估分约 1400~1500
```

若给出区间，端点也必须使用上述合法档位。

## 必须额外给出的结论

### 1. 主导难点

指出本题主要难在哪里，例如：

- 算法 / 模板门槛
- 结论发现难度
- 思维转化
- 实现与边界
- 题意阅读与建模
- 错误方向诱导
- 赛时验证困难

### 2. 赛时波动

评价该题在赛时的体感不稳定程度：

- 低：难度主要来自稳定可训练内容
- 中：存在一定观察，但可通过系统推导得到
- 高：短代码、强观察、关键结论型题，想到和想不到差异巨大
- 极高：结论隐蔽、错路多、样例误导，赛时表现高度不稳定

### 3. 训练价值

简要说明适合训练什么，例如：

- 数论观察
- 构造
- 贪心交换论证
- 状态设计
- 树形 DP
- 实现能力
- 题意建模
- 打表找规律
- 反例构造
- 复杂边界处理
- minimax 建模

## 注意事项

1. 不使用官方 rating 作为打分依据。
2. 官方 rating 只能作为事后对照，不能反向影响评分。
3. 必须同时给出公共难度和目标人群体感难度。
4. 目标人群体感难度不能反向修改公共难度。
5. 短代码题不能因为实现短就简单化，也不能因为结论表述陌生就过度拔高。
6. 板子题必须区分“会板子的人”和“不会板子的人”的体感差异。
7. 平均分可能掩盖单点门槛，因此必须应用瓶颈规则。
8. 对强观察题，应额外说明赛时波动。
9. 对打表规律题，应评估规律可发现性。
10. 若多个维度来自同一个核心观察，必须进行难点去重。
11. 若存在多个赛时合理做法，应按最简单可想到正解评分。
12. 必须区分赛时 AC 难度、严格证明难度、实现落地难度。
13. 若题目数据强弱会影响通过难度，应分别说明理论正解难度和赛时通过难度。
14. CF 估分必须落在合法档位：`800, 900, ..., 3500, 3500+`。
