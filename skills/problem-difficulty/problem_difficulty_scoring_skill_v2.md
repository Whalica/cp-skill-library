# 题目难度评分 Skill v2

## 目标

用于脱离官方 rating，对算法竞赛题目进行结构化难度评价，并给出：

- 可解释的结构分；
- 赛时修正后的最终难度；
- 主导难点；
- 赛时波动；
- 训练价值；
- CF 难度档位估计。

本 Skill 的目标不是复刻官方 rating，而是尽量接近“目标选手群体的平均赛时体感难度”。

---

## 评分范围

所有基础维度分数均为 `0.0 ~ 10.0`，保留一位小数。

分数含义：

- `0.0`：几乎没有难度
- `2.0`：较低难度
- `5.0`：中等难度
- `8.0`：较高难度
- `10.0`：极高难度

最终分也控制在 `0.0 ~ 10.0` 内，保留一位小数。

---

## 目标人群

评分前应默认选择一个目标人群。若用户没有指定，默认使用：

```text
CF 1400~1600 左右的普通竞赛选手
```

可选目标人群示例：

- 普通 Div.2 参赛者
- CF 1200~1400 选手
- CF 1400~1600 选手
- CF 1600~1900 选手
- 区域赛队员
- 已系统训练过对应专题的选手

模板题或套路题必须说明：

- 模板熟练者体感
- 模板不熟者体感

---

## 基础评价维度

### 1. 算法 / 模板门槛

评价是否需要专门算法、数据结构或固定模板。

没有明显模板时，不纳入基础分，不给 `0`。

参考：

- `0 ~ 2`：只用基础语法、排序、gcd、前缀和、简单枚举等
- `3 ~ 5`：常见基础算法，如二分、BFS、DFS、简单 DP、并查集、Fenwick
- `6 ~ 8`：较复杂模板，如线段树、树形 DP、最短路变形、字符串算法、网络流基础
- `9 ~ 10`：高门槛模板，如后缀自动机、点分治、支配树、高级图论、高级数据结构

注意：

- 裸模板题要区分“会模板的人”和“不会模板的人”的体感差异。
- 若模板只是辅助，不能把模板门槛当作主导难点。
- 若题目有无模板解，应按赛时合理可想到的最简单正解评分。

---

### 2. 结论发现难度

评价关键性质、观察、构造方式在赛时被发现的难度。

此项优先评价“发现成本”，而不是结论本身是否少见。

参考：

- `0 ~ 2`：结论几乎直接可见，样例或题面强提示
- `3 ~ 5`：需要手推、画图、试小样例，但较自然
- `6 ~ 8`：需要针对性推导，不容易通过普通小样例看出
- `9 ~ 10`：核心结论非常隐蔽，接近“想不到就做不了”

重要区分：

```text
结论陌生 ≠ 结论难发现
```

若一个结论看起来陌生，但能通过 3~5 个小样例自然发现，则本项不应给高分。

---

### 3. 思维转化难度

评价从题面到正解需要经过多少层转化，以及逻辑推理链是否自然。

参考：

- `0 ~ 2`：题面几乎直接给做法
- `3 ~ 5`：需要一到两层常规转化
- `6 ~ 8`：需要多层推理、反向思考或关键建模
- `9 ~ 10`：推理链很深，切入点极难自然想到

注意：

- 若本项打高的原因与“结论发现难度”完全相同，应进行难点去重。
- 单点观察题不能在结论发现、思维转化、建模三个维度中重复计费。

---

### 4. 实现与边界风险

评价代码实现成本，而不只看代码长度。

考虑因素：

- 代码量
- 状态维护复杂度
- 函数数量
- 分类讨论数量
- 下标处理
- 取模 / 溢出
- 多测清空
- 特殊边界
- 调试成本
- 出错概率

参考：

- `0 ~ 2`：几行判断或简单循环，几乎无坑
- `3 ~ 5`：普通实现，有少量状态或边界
- `6 ~ 8`：实现较长，状态多，容易写错
- `9 ~ 10`：大型模拟、复杂数据结构、多层分类讨论，debug 成本很高

强制规则：

- 若实现复杂度 `>= 7.0`，最终分通常不应低于 `5.5`。
- 若实现复杂度 `>= 8.0` 且边界密集，最终分通常不应低于 `6.0`。
- 几何、交互、大模拟、复杂构造应额外关注验证难度。

---

### 5. 题意阅读与建模难度

评价理解题意、整理操作、抽象状态、建立数学或算法模型的难度。

可拆成两部分思考：

- 题面阅读难度：题面是否长、绕、角色多、时序复杂、翻译腔严重
- 模型抽象难度：是否难以把题面转化成可计算模型

参考：

- `0 ~ 2`：题意直接，模型显然
- `3 ~ 5`：题面较长或状态需要整理
- `6 ~ 8`：操作复杂，抽象模型不明显
- `9 ~ 10`：题意本身非常绕，建模是主要门槛

注意：

- 若题面绕但算法简单，应体现阅读成本。
- 若题面短但模型抽象，也应体现建模成本。
- 若建模难点与核心观察完全相同，应进行难点去重。

---

### 6. 错误方向诱导性 / 试错成本

评价题目是否容易把选手引向错误做法，以及排除错误方向的成本。

参考：

- `0 ~ 2`：做法方向自然，错路少
- `3 ~ 5`：有一些常见误区
- `6 ~ 8`：错路很多，容易写假算法
- `9 ~ 10`：正解反直觉，错误方向极强

典型加分场景：

- 看起来能贪心，但贪心全错
- 看起来能 DP，但状态爆炸
- 看起来要数据结构，其实是结论
- 样例弱，假做法容易过样例
- 存在大量伪规律
- 正解很短但方向极不自然

此项一般纳入基础分。若题目几乎没有误导性，可给低分纳入，而不是不纳入。

## 基础分计算

对纳入的基础维度加权平均。

默认权重：

| 维度 | 权重 |
|---|---:|
| 算法 / 模板门槛 | 0.20 |
| 结论发现难度 | 0.25 |
| 思维转化难度 | 0.20 |
| 实现与边界风险 | 0.20 |
| 题意阅读与建模难度 | 0.15 |
| 错误方向诱导性 / 试错成本 | 0.15 |

若某项“不纳入”，则去掉该项权重，并对剩余权重归一化。

公式：

```text
基础分 = Σ(维度分 × 维度权重) / Σ(纳入维度权重)
```

注意：

- 不再使用简单算术平均作为唯一规则。
- 仍需列出所有维度分数，保证可解释性。
- 若用户要求沿用旧版平均法，可额外给出旧版平均分作为对照。

## 赛时修正项

基础分反映题目结构难度，最终分需要根据赛时体感修正。

修正项可以为正或负，最终分计算：

```text
最终分 = clamp(基础分 + 修正项总和, 0.0, 10.0)
```

修正项总和一般控制在 `-2.0 ~ +2.0`，极端情况可突破，但必须解释。

---

### 1. 规律可发现性修正

评价核心规律是否容易通过小数据、打表、画图、手推样例发现。

| 情况 | 修正 |
|---|---:|
| 3~5 个小样例即可明显发现规律 | `-0.8 ~ -1.2` |
| 小样例有一定提示，但不稳定 | `-0.3 ~ -0.6` |
| 小样例无明显提示 | `0` |
| 小样例明显误导 | `+0.3 ~ +0.8` |

典型适用：

- XOR 周期题
- 模运算循环题
- 小规模递归生成题
- 构造规律题
- 可打表归纳题

---

### 2. 约束提示强度修正

评价数据范围是否直接暗示做法。

| 情况 | 修正 |
|---|---:|
| 约束非常明显地提示正解方向 | `-0.5 ~ -1.0` |
| 约束有一定提示 | `-0.2 ~ -0.5` |
| 约束无明显提示 | `0` |
| 约束具有迷惑性 | `+0.3 ~ +0.8` |

例：

- `n <= 20` 强提示状压 / 折半
- `k <= 30` 且出现 `2^k` 强提示找规律或按位分析
- `值域 <= 1e6` 强提示筛法 / 频率统计
- `sum n <= 2e5` 提示线性或 `n log n`

---

### 3. 难点去重修正

若多个维度的高分来自同一个核心难点，应下调。

| 情况 | 修正 |
|---|---:|
| 两个维度明显重复计费 | `-0.3 ~ -0.6` |
| 三个及以上维度来自同一核心观察 | `-0.6 ~ -1.0` |
| 多个难点彼此独立 | `0` |

示例：

若题目唯一难点是“发现某个不变量”，则不能同时在：

- 结论发现难度
- 思维转化难度
- 题意建模难度

三个维度都给高分。

---

### 4. 多解路径修正

评价是否存在更简单、赛时合理可想到的替代做法。

| 情况 | 修正 |
|---|---:|
| 有明显更简单做法 | `-0.5 ~ -1.0` |
| 有暴力 / 打表 / 对拍辅助发现 | `-0.3 ~ -0.8` |
| 正解路径唯一且很窄 | `+0.3 ~ +0.8` |
| 只看到复杂官方解，民间解未知 | 暂不修正，并说明不确定 |

原则：

```text
评分应基于赛时合理可想到的最简单正解，而不是最正式、最优雅或官方给出的做法。
```

---

### 5. 可验证性 / 对拍难度修正

评价猜到做法后，选手是否容易通过暴力、随机对拍、checker 验证正确性。

| 情况 | 修正 |
|---|---:|
| 很容易写暴力对拍 | `-0.3 ~ -0.6` |
| 普通 | `0` |
| 很难验证 | `+0.3 ~ +0.8` |

典型难验证题：

- 几何
- 交互
- 浮点
- 复杂构造
- 复杂图论
- 输出方案合法性难检查

---

### 6. 样例提示修正

评价样例是否显著降低或增加赛时难度。

| 情况 | 修正 |
|---|---:|
| 样例强烈提示正解 | `-0.3 ~ -0.8` |
| 样例覆盖主要边界 | `-0.2 ~ -0.4` |
| 样例普通 | `0` |
| 样例弱或误导 | `+0.2 ~ +0.6` |

---

### 7. 数据强弱修正

若评价目标是“赛时通过难度”，可以考虑数据强弱。

| 情况 | 修正 |
|---|---:|
| 数据明显弱，假做法可过 | 赛时通过难度下调 |
| pretest 弱但 system test 强 | 说明赛时风险，不直接下调理论难度 |
| 数据强，卡边界充分 | 实现与边界风险上调 |
| 数据未知 | 不修正 |

必须区分：

- 理论正解难度
- 赛时通过难度

---

## 瓶颈规则

平均或加权平均可能掩盖单点门槛，因此需要瓶颈规则。

### 1. 单点门槛下限

若存在不可绕过的核心门槛，最终分通常不应低于：

```text
核心门槛分 - 0.5
```

例如：

- 唯一关键观察难度为 `8.0`
- 实现复杂度只有 `1.5`

最终分仍不应被平均拉到 `4.0` 以下。

---

### 2. 短代码规律题上限

若同时满足：

- 无明显算法 / 模板门槛
- 实现短
- 边界少
- 核心规律可通过 3~5 个小样例发现
- 没有强误导方向

则最终分通常不应超过：

```text
4.5
```

若规律极其明显，可进一步限制在：

```text
3.5 ~ 4.0
```

---

### 3. 实现题下限

若题目主思路常规，但实现复杂度极高：

- 实现复杂度 `>= 7.0`：最终分通常不低于 `5.5`
- 实现复杂度 `>= 8.0` 且边界密集：最终分通常不低于 `6.0`

适用：

- 大模拟
- 多分类构造
- 复杂数据结构维护
- 几何
- 交互
- 复杂 DP 状态维护

---

## 难度类型分离

输出时应尽量区分：

```text
赛时 AC 难度
严格证明难度
实现落地难度
题解表达难度
```

说明：

- 赛时 AC 难度：选手在比赛中做出并通过的难度，最终 CF 估分主要参考它。
- 严格证明难度：把正确性完整讲清楚的难度。
- 实现落地难度：知道做法后写出并调对的难度。
- 题解表达难度：把题目写成清楚题解的难度。

对于“猜规律可 AC，但证明较难”的题，应避免高估赛时 AC 难度。

## CF 估分规则

### 1. 连续估分

先用最终分给出连续估计：

```text
连续估分 = 800 + 200 × 最终分
```

结果可以保留到个位，用于中间计算。

---

### 2. CF 档位化

最终展示时必须按 CF 实际题目难度档位化。

合法档位为：

```text
800, 900, 1000, 1100, 1200, 1300, 1400, 1500,
1600, 1700, 1800, 1900, 2000, 2100, 2200, 2300,
2400, 2500, 2600, 2700, 2800, 2900, 3000, 3100,
3200, 3300, 3400, 3500, 3500+
```

档位化规则：

1. 若连续估分 `< 850`，档位为 `800`。
2. 若连续估分在 `[850, 3450)`，四舍五入到最近的整百。
3. 若连续估分在 `[3450, 3550)`，档位为 `3500`。
4. 若连续估分 `>= 3550`，档位为 `3500+`。

可同时给出区间：

```text
CF 估分约 1400~1500
```

若给出区间，端点也必须使用上述合法档位。

---

### 3. 档位校准表

线性映射只是初估，最终应结合题型修正。

| 最终分 | 常见 CF 档位参考 |
|---:|---:|
| `0.0 ~ 1.5` | `800 ~ 1000` |
| `1.5 ~ 2.5` | `1000 ~ 1200` |
| `2.5 ~ 3.5` | `1200 ~ 1400` |
| `3.5 ~ 4.5` | `1400 ~ 1600` |
| `4.5 ~ 5.5` | `1600 ~ 1800` |
| `5.5 ~ 6.5` | `1800 ~ 2100` |
| `6.5 ~ 7.5` | `2100 ~ 2400` |
| `7.5 ~ 8.5` | `2400 ~ 2700` |
| `8.5 ~ 9.5` | `2700 ~ 3200` |
| `9.5 ~ 10.0` | `3200 ~ 3500+` |

若线性估分与档位校准表冲突，应优先给出解释，并以赛时体感为准。

## 必须额外给出的结论

### 1. 主导难点

指出本题主要难在哪里，例如：

- 算法 / 模板门槛
- 结论发现难度
- 思维转化
- 实现与边界
- 题意阅读与建模
- 错误方向诱导
- 赛时验证困难

若有多个主导难点，可以并列。

---

### 2. 赛时波动

评价该题在赛时的体感不稳定程度：

- 低：难度主要来自稳定可训练内容，如模板或常规 DP
- 中：存在一定观察，但可通过系统推导得到
- 高：短代码、强观察、关键结论型题，想到和想不到差异巨大
- 极高：结论隐蔽、错路多、样例误导，赛时表现高度不稳定

---

### 3. 训练价值

简要说明适合训练什么，例如：

- 数论观察
- 构造
- 贪心交换论证
- 状态设计
- 树形 DP
- 实现能力
- 题意建模
- 打表找规律
- 反例构造
- 复杂边界处理
- minimax 建模

---

## 常见题型修正表

| 题型 | 原体系易出现的问题 | v2 修正方式 |
|---|---|---|
| 打表规律题 | 高估 | 降低结论发现难度，加入规律可发现性负修正 |
| 短代码不变量题 | 高估 | 难点去重，避免多维度重复计费 |
| 单点神仙观察题 | 低估 | 使用瓶颈下限 |
| 大模拟 | 低估 | 提高实现与边界风险，使用实现题下限 |
| 模板裸题 | 人群差异大 | 输出模板熟练者 / 不熟练者体感 |
| 几何题 | 低估 | 增加验证难度、精度风险、边界风险 |
| 构造题 | 双向误差 | 看规律可发现性、多解路径、验证难度 |
| 贪心陷阱题 | 低估 | 增加错误方向诱导性 |
| 题面长但算法简单 | 低估 | 拆出题意阅读成本 |
| 官方解复杂但民间解简单 | 高估 | 按赛时合理最简单正解评分 |
| 证明难但猜可 AC | 高估 | 区分赛时 AC 难度与严格证明难度 |
| 数据弱题 | 理论与通过难度混淆 | 分开理论正解难度与赛时通过难度 |

## 注意事项

1. 不使用官方 rating 作为打分依据。
2. 官方 rating 只能作为事后对照，不能反向影响评分。
3. 短代码题不能因为实现短就简单化，也不能因为结论表述陌生就过度拔高。
4. 板子题必须区分“会板子的人”和“不会板子的人”的体感差异。
5. 平均分可能掩盖单点门槛，因此必须应用瓶颈规则。
6. 对强观察题，应额外说明赛时波动。
7. 对打表规律题，应评估规律可发现性。
8. 若多个维度来自同一个核心观察，必须进行难点去重。
9. 若存在多个赛时合理做法，应按最简单可想到正解评分。
10. 必须区分赛时 AC 难度、严格证明难度、实现落地难度。
11. 若题目数据强弱会影响通过难度，应分别说明理论正解难度和赛时通过难度。
12. CF 估分必须落在合法档位：`800, 900, ..., 3500, 3500+`。
