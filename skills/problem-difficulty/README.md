# README：如何使用题目难度评分 Skill

本文件夹包含：

- `problem_difficulty_scoring_skill_v3.md`：题目难度评分 Skill 主文件
- `README.md`：本说明文件

## 这个 Skill 做什么

它用于评价算法竞赛题目的难度，并且同时输出两类结果：

1. **公共难度**
   - 题目本身大约是什么难度。
   - 最终会映射到 CF 档位，例如 `1300`、`1600`、`1900`、`3500+`。

2. **目标人群体感难度**
   - 指定水平的选手做这题时感觉多难。
   - 例如 CF 4000 选手做 1600 题，体感可能只有 `0.5/10`，但公共难度仍然是 `1600`。

核心原则：

```text
公共难度 ≠ 目标人群体感难度
```

## 如何修改目标人群

打开 `problem_difficulty_scoring_skill_v3.md`，找到这一段：

```yaml
target_group:
  name: "CF 1400~1600 左右普通竞赛选手"
  rating: 1400~1600
```

把 `rating` 改成你想评价的目标人群 rating 即可。

例如：

### 改成 CF 1500 选手

```yaml
target_group:
  name: "CF 1500 左右普通竞赛选手"
  rating: 1500
```

### 改成 CF 2200 选手

```yaml
target_group:
  name: "CF 2200 左右普通竞赛选手"
  rating: 2200
```

### 改成 CF 4000 选手

```yaml
target_group:
  name: "CF 4000 左右普通竞赛选手"
  rating: 4000
```

## 修改 rating 会影响什么

会影响：

- 目标人群体感分；
- 目标人群赛时体感描述；
- 模板熟练度判断；
- “这题对该目标人群是否是热身题 / 常规题 / 难题”的结论。

不会直接影响：

- 公共难度；
- 公共 CF 估分；
- 题目客观结构评价。

例如一题公共难度是 `1900`：

| 目标人群 | 目标人群体感 |
|---|---:|
| CF 1400 | 很难 |
| CF 1900 | 正常匹配 |
| CF 2600 | 偏简单 |
| CF 4000 | 热身题 |

但这题的公共 CF 估分仍然应是 `1900` 左右。

## 推荐用法

向模型提问时可以这样写：

```text
按 problem_difficulty_scoring_skill_v3.md 评价这题：
<题目链接>
目标人群 rating 使用 1400~1600
```

或者：

```text
按 v3 skill 评价这题，公共难度和目标人群体感都要给出，目标人群为 CF 2200。
```

## CF 档位

公共 CF 估分必须落在以下档位：

```text
800, 900, 1000, 1100, 1200, 1300, 1400, 1500,
1600, 1700, 1800, 1900, 2000, 2100, 2200, 2300,
2400, 2500, 2600, 2700, 2800, 2900, 3000, 3100,
3200, 3300, 3400, 3500, 3500+
```

## 常见误区

### 误区 1：目标人群很强，所以公共难度也要变低

错误。

公共难度评价题目本身，不随目标人群 rating 直接变化。

### 误区 2：公共难度低，所以对所有人都简单

错误。

对弱于题目公共难度很多的选手，体感仍然可能很高。

### 误区 3：只有一个分数就够了

不够。

建议始终输出：

- 公共最终分；
- 公共 CF 估分；
- 目标人群体感分；
- 主导难点；
- 赛时波动；
- 训练价值。

## 最小输出格式

如果想要精简，可以只保留：

```markdown
## 结论

公共难度：x.x/10，公共 CF 估分约 xxxx  
目标人群：CF yyyy  
目标人群体感：x.x/10

主导难点：...
赛时波动：...
训练价值：...
```
