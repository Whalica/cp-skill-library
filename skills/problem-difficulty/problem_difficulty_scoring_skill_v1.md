# 题目难度评分 Skill

## 目标

用于脱离官方 rating，对算法竞赛题目进行结构化难度评价，并给出可解释的总分、主导难点、赛时波动与 CF 估分。

## 评分范围

所有分数均为 `0.0 ~ 10.0`，保留一位小数。

分数含义：

- `0.0`：几乎没有难度
- `2.0`：较低难度
- `5.0`：中等难度
- `8.0`：较高难度
- `10.0`：极高难度

## 评价维度

### 1. 板子难度

判断是否需要专门算法、数据结构或固定模板。

没有明显板子时，不纳入总分。

参考：

- `0 ~ 2`：只用基础语法、排序、gcd、前缀和等
- `3 ~ 5`：常见基础算法，如二分、BFS、DFS、简单 DP、并查集、Fenwick
- `6 ~ 8`：较复杂模板，如线段树、树形 DP、最短路变形、字符串算法、网络流基础
- `9 ~ 10`：高门槛模板，如后缀自动机、点分治、支配树、高级图论、高级数据结构

### 2. 结论陌生度

评价关键性质、结论、观察是否常见，是否容易由经验直接推出。

分数越高，表示结论越不通识、越隐蔽。

参考：

- `0 ~ 2`：常识级结论，几乎直接可见
- `3 ~ 5`：常见竞赛结论，需要一定经验
- `6 ~ 8`：不太常见，需要针对性推导或较强观察
- `9 ~ 10`：非常隐蔽，接近“想不到就做不了”的核心猜想

### 3. 实现复杂度

评价代码实现成本，而不只看代码长度。

考虑因素：代码量、状态维护、函数数量、边界条件、分类讨论、下标处理、取模/溢出、调试成本、出错概率。

参考：

- `0 ~ 2`：几行判断或简单循环，几乎无坑
- `3 ~ 5`：普通实现，有少量状态或边界
- `6 ~ 8`：实现较长，状态多，容易写错
- `9 ~ 10`：大型模拟、复杂数据结构、多层分类讨论，debug 成本很高

### 4. 思维层级

评价从题面到做法需要经过多少层转化，以及逻辑推理难度。

考虑因素：建模转化次数、是否需要反向思考、是否依赖关键观察、错误方向诱导性、灵光乍现依赖程度。

参考：

- `0 ~ 2`：题面几乎直接给做法
- `3 ~ 5`：需要一到两层常规转化
- `6 ~ 8`：需要多层推理或关键观察
- `9 ~ 10`：推理链很深，关键切入点极难自然想到

### 5. 题意建模难度

评价理解题意、抽象状态、建立数学或算法模型的难度。

没有明显建模难点时，可以纳入低分，也可以说明其非主导难点。

参考：

- `0 ~ 2`：题意直接，模型显然
- `3 ~ 5`：题面较长或状态需要整理
- `6 ~ 8`：操作复杂，抽象模型不明显
- `9 ~ 10`：题意本身非常绕，容易误读，建模是主要门槛

## 总分规则

只对“纳入评价”的维度取平均。

公式：

```text
总分 = 纳入维度分数之和 / 纳入维度数量
```

保留一位小数。

板子不存在时，不给 `0`，而是标记为“不纳入”。

## CF 估分映射

使用线性映射：

```text
CF估分 = 800 + 200 × 总分
```

然后按常见 CF 档位四舍五入到整百或半百。

参考：

- `0.0` → `800`
- `4.0` → `1600`
- `5.0` → `1800`
- `6.0` → `2000`
- `8.0` → `2400`
- `10.0` → `2800`

## 必须额外给出的结论

### 主导难点

指出本题主要难在哪里，例如：

- 板子门槛
- 结论陌生度
- 实现复杂度
- 思维层级
- 题意建模难度

若有多个主导难点，可以并列。

### 赛时波动

评价该题在赛时的体感不稳定程度：

- 低：难度主要来自稳定可训练内容，如模板或常规 DP
- 中：存在一定观察，但可通过系统推导得到
- 高：短代码、强观察、关键结论型题，想到和想不到差异巨大

### 训练价值

简要说明适合训练什么，例如：

- 数论观察
- 构造
- 贪心交换论证
- 状态设计
- 树形 DP
- 实现能力
- 题意建模

## 注意事项

1. 不使用官方 rating 作为打分依据。
2. 官方 rating 只能作为事后对照，不能反向影响评分。
3. 短代码题不能因为实现短就简单化，应单独看结论与思维。
4. 板子题要区分“会板子的人”和“不会板子的人”的体感差异。
5. 平均分可能掩盖单点门槛，因此必须给出主导难点。
6. 对强观察题，应额外说明赛时波动。
